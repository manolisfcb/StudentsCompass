# StudentsCompass Capstone Product Documentation

This folder is the working documentation home for the StudentsCompass capstone.
Use it to collect product decisions, technical evidence, implementation notes,
screenshots, test results, and architecture records before assembling the final
academic submission.

## Current Documentation

- `phase_1_status.md`: current Phase 1 product and capstone status.
- `service_architecture.md`: backend service-domain structure and import policy.

## Documentation Rule

Every substantial capstone/product change should add a short note here covering:

- what changed;
- why it matters for StudentsCompass as a product;
- what evidence proves it works;
- what remains as a limitation or next step.
