# StudentsCompass Capstone Product Documentation

This folder is the working documentation home for the StudentsCompass capstone.
Use it to collect product decisions, technical evidence, implementation notes,
screenshots, test results, and architecture records before assembling the final
academic submission.

## Current Documentation

- `phase_1_status.md`: current Phase 1 product and capstone status.
- `phase_2_plan.md`: Phase 2 scope, implementation plan, and acceptance
  criteria.
- `phase_4_completion.md`: final Phase 4 productization status, delivered
  files, API surface, UI behavior, runbook, and validation evidence.
- `phase_4_skill_extraction.md`: Phase 4 skill extraction, normalization, and
  evaluation evidence.
- `phase_4_1_product_hardening.md`: product hardening for resume skill review,
  expanded catalog, and aggregate dataset evaluation.
- `phase_5_skill_gap_scoring.md`: Phase 5 scoring method for ranking missing
  skills by role weight minus student evidence.
- `phase_6_optimization_model.md`: formal Phase 6 mathematical optimization
  model, including variables, objective, constraints, CP-SAT implementation
  notes, explainability, and validation plan.
- `phase_6_validation_evidence.md`: synthetic Phase 6 evidence suite for
  CP-SAT constraints, objective behavior, sequencing, and endpoint regression.
- `phase_7_baseline_evaluation.md`: Phase 7 baseline comparison plan and
  implementation evidence for CP-SAT versus cheapest, highest-rated,
  similarity-only, heuristic, and seeded random routes.
- `matching_methodology.md`: exact and semantic matching methodology.
- `optimization_methodology.md`: learning route optimization methodology.
- `service_architecture.md`: backend service-domain structure and import policy.

## Documentation Rule

Every substantial capstone/product change should add a short note here covering:

- what changed;
- why it matters for StudentsCompass as a product;
- what evidence proves it works;
- what remains as a limitation or next step.
